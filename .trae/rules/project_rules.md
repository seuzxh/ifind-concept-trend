# 项目规则文档

## 1. 虚拟环境管理

- **环境管理工具**: conda
- **虚拟环境名称**: concept-trend
- **Python 版本**: 3.10

## 2. 环境变量

- **配置文件**: 项目根目录 `.env`（参考 `.env.example`）
- **敏感令牌禁止写入代码或版本控制**
- 必须在项目根目录创建 `.env` 文件，`.env.example` 仅作为模板提交

### 必需环境变量

| 变量名 | 用途 | 来源 |
|--------|------|------|
| `IFIND_REFRESH_TOKEN` | ifind 量化 API 鉴权令牌 | 同花顺量化平台获取 |

### 环境变量使用规范

- 通过 `os.getenv()` 读取环境变量，提供合理的默认值
- 环境变量名使用全大写 + 下划线分隔（如 `IFIND_REFRESH_TOKEN`）
- 新增环境变量时必须同步更新 `.env.example`
- `.env` 文件必须在 `.gitignore` 中（禁止提交敏感信息）
- 配置模板文件为 `config/config.yaml.template`，实际配置为
  `config/config.yaml`（同样禁止提交）

## 3. 任务执行流程

- **每完成一个 task/subtask 后**，必须先验证（运行测试/脚本确认无误），再 `git commit` 提交变更，然后才能进行下一步
- 验证方式：运行对应的验证脚本，或通过 Python 交互确认代码行为正确
- 提交信息格式：`feat/fix/docs: 简要描述（Task X.X）`
- 若验证失败，必须修复后再提交，不得提交未通过验证的代码

## 4. README 维护

- **每次完成任务后**，必须检查项目根目录的 `README.md` 是否需要更新
- 当任务涉及以下变更时，应主动更新 README：
  - 新增或修改了项目功能、模块、流程
  - 变更了项目结构、配置方式、环境依赖
  - 新增或修改了命令行接口、脚本入口
  - 变更了数据模型、数据库表结构、API 接口
- 若 README 已与当前代码状态一致，则无需修改
- 更新 README 时保持内容简洁准确，与代码实际行为一致
